## When To Use

- Create structured and validated forms with consistent styling
- Client-side validation with user-friendly error messages
- Accessible form controls with proper ARIA attributes
- Manage form state efficiently with React Hook Form

## Components

### Form

Re-export of React Hook Form's `FormProvider`. Wraps your form and provides context to all child components.

### FormField

Wraps React Hook Form's `Controller`. Connects a form field to the form context.

| Prop | Type | Description |
|------|------|-------------|
| `control` | `Control` | Form control from `useForm()` |
| `name` | `string` | Field name matching your schema |
| `render` | `({ field }) => ReactNode` | Render function for the field |

### FormItem

Container for a single form field. Provides layout (`grid gap-2`) and context for label/control/message linking.

### FormLabel

Styled label that auto-links to its `FormControl` via `htmlFor`. Sets `data-error` when validation fails.

### FormControl

Wraps the actual input element via Radix `Slot`. Passes down:

- `id` — links to `FormLabel`
- `aria-describedby` — links to `FormDescription` and `FormMessage`
- `aria-invalid` — set to `true` when field has an error
- `shaky-effect` class — applied when the form has only 1 field and validation fails

### FormDescription

Helper text rendered below the input. Linked via `aria-describedby`.

### FormMessage

Displays the validation error message. Uses `text-error-border-2` color.

## Error Styling

Error border and shadow styles are applied via CSS `has()` selector on the `Input` wrapper:

```
has-[input[aria-invalid=true]]:border-error-border-2
has-[input[aria-invalid=true]]:shadow-[0_0_4px_1px_rgb(var(--error-border-2))]
```

This ensures the outer border of the `Input` component reacts to `aria-invalid` on the inner `<input>` element, rather than styling the inner input directly.

## Accessibility

- All form controls are linked to their labels via `id`/`htmlFor`
- Error messages are announced via `aria-describedby`
- Invalid fields are marked with `aria-invalid`
- Shaky animation provides visual feedback on single-field forms
