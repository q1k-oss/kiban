## When To Use

- Render icons dynamically using a string name (from API, config, etc.)
- Combine Lucide icons with custom SVG icons in one component
- Fallback icon for missing entries — no runtime crashes
- Autocomplete and type safety for icon names

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `iconName` | `string` | — | Name of the icon. Supports Lucide names and custom icon names (prefixed with `custom-`) |
| `size` | `number` | `20` | Icon size in pixels |
| `strokeWidth` | `number` | `1.5` | Stroke width for the icon |
| `className` | `string` | — | Additional CSS classes |
| `source` | `"lucide" \| "custom"` | `"lucide"` | Icon source. Use `"custom"` for project-specific SVG icons |

## Usage

```tsx
// Lucide icon (default)
<AppIcon iconName="check" />

// Lucide icon with custom size
<AppIcon iconName="settings" size={32} strokeWidth={2} />

// Custom icon
<AppIcon iconName="custom-q1k-logo" source="custom" />

// Custom icon with styling
<AppIcon iconName="custom-ai-stars" source="custom" size={40} className="text-primary" />
```

## Custom Icons

Custom icons are registered in `components/icons/index.tsx`. To add a new custom icon, create an SVG component and add it to the `CustomIcons` registry with a `custom-` prefixed key.
