### When To Use

Use **AppIcon** when you need to:

- Render icons dynamically using a string value
- Support icons coming from backend APIs or configurations
- Combine **Lucide icons** with **custom SVG icons**
- Avoid runtime crashes when an icon is missing
- Enable autocomplete and type safety for icon names
- Future-proof your UI when icons may increase or change

`AppIcon` provides a single, scalable solution for resolving icons from multiple sources while maintaining clean and maintainable code.

---

### Features

- Autocomplete support for icon names (Lucide + Custom)
- Backend-safe string handling
- Fallback icon for missing entries
- Centralized icon resolution logic
